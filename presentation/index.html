What is Kotlin?

Statically typed programming language

Developed by JetBrains

Compiles to JVM bytecode or JavaScript

Access to all existing Java frameworks and libraries

A little bit of history...

July 2011: JetBrains unveiled Project Kotlin

February 2012: Open sourced under Apache 2 license

February 15, 2016: Release of v1.0

Google I/O 2017: Kotlin official language for Android

March 1, 2017: v1.1 released

November 28, 2017: v1.2 released

October 29, 2018 v1.3 released

Kotlin aims to be...

Object-oriented language with functions as first class citizens

Readable, concise and expressive

Safety through null-types and immutability

Strong focus on interoperability with Java

Safer code by promoting immutability and a functional approach

Nullable values

By default everything is non-nullable 


val value: String = null // ❌ compiler error

Everything is an object in Kotlin

Nullable is part of the type


val value: String? = null // ✅

Protects you from mistakenly operating on nullable types 


val person = findPerson() // Person?
println(person.name) // ❌ compiler error

Dealing with nullability

Check for null or use safe call operator 


// checking for null
val person = findPerson() // Person?
if (person != null) { // cast to non-nullable after null check
    println(person.name)
}

// safe call operator
println(person?.name ?: "Unknown")

Can't I just skip all that?

If you’re really a fan of NPEs... 


val person = findPerson() // Person?
println(person!!.name)

Type checks and automatic casts


fun getStringLength(obj: Any): Int? {
    if (obj is String) { // Automatically cast to `String`
        return obj.length
    }
    return null
}

is is similar to instanceof in Java

Progression

When expression

Switch statement on steroids 


fun describe(obj: Any) = when (obj) {
    0, 1 -> "Zero or one"
    in 5..10 -> "In the range of five to ten"
    "Hello" -> "Greeting"
    is Person -> "Hi $obj.name"
    !is String -> "Not a string"
    else -> "Unknown"
}

Open modifier and inheritance

Classes are by default final

Use the open modifier to allow inheriting it


open class Animal {
    fun run() = println("Running...")
    open fun makeSound() = println("Herp")
}
class Cat : Animal() {
    override fun makeSound() = println("Mijauwkes")
}

Combining properties between parent and child


open class Animal(val name: String)

class Cat(name: String) : Animal(name) {
    constructor(name: String, hasFur: Boolean) : this(name) {
        println("Cat ${if (hasFur) "has" else "doesn't have"} fur")
    }
}

val cat = Cat("Mr. Bigglesworth", hasFur = true)
